# VoiceScope Mobile (Android)

Capacitor wrapper around the shared `client/` React app. Same UI as the
desktop Electron version, but the runtime is Android WebView and platform
APIs are accessed through Capacitor plugins instead of Electron IPC.

## Architecture

```
client/  (React + Tailwind, shared with desktop)
   │
   │  npm run build  →  client/dist/
   │
   ▼
mobile/  (this directory)
   │
   ├── capacitor.config.ts   ← points webDir at ../client/dist
   ├── android/              ← generated by `cap add android`
   │     └── app/src/main/   ← Android manifest, icons, etc.
   │
   └── runtime calls go to:
         ├── Cloudflare Worker (managed mode, default)
         ├── Local LLM endpoint (Ollama on PC/NAS, companion app)
         └── Direct provider APIs (own-key mode)
```

## First-time bootstrap

The `android/` directory is **generated**, not committed. After cloning,
the maintainer runs:

### 1. Install dependencies (root + client + mobile)

```powershell
cd Z:\projects\voicescape
npm install
cd client
npm install
cd ..\mobile
npm install
```

### 2. Build the client

```powershell
cd Z:\projects\voicescape\client
npm run build
```

### 3. Initialize Capacitor + add Android

From `Z:\projects\voicescape\mobile`:

```powershell
npx cap init "VoiceScope" "com.bamipeko.voicescape" --web-dir "../client/dist"
npx cap add android
npx cap sync android
```

This creates the `android/` directory with a Gradle project Android Studio
can open.

### 4. Open in Android Studio

```powershell
npx cap open android
```

Wait for Gradle sync (~5 min first time). Then:

- File → Project Structure → SDK Location: ensure SDK 34 is installed
- Run → Run 'app': pick your physical device or emulator

### 5. Required Android permissions

Add to `android/app/src/main/AndroidManifest.xml` inside `<manifest>`:

```xml
<uses-permission android:name="android.permission.RECORD_AUDIO" />
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
                 android:maxSdkVersion="32" />
<uses-permission android:name="android.permission.READ_MEDIA_AUDIO" />
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_MICROPHONE" />
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
```

The `FOREGROUND_SERVICE_MICROPHONE` permission is required by Android 14+
for background recording — without it the OS will kill recording when the
app is backgrounded.

## Daily dev loop

### Option A: Live reload via Vite dev server (fast)

Terminal 1 — Vite dev server:
```powershell
cd Z:\projects\voicescape\client
npm run dev
```
Note the LAN IP (Vite prints e.g. `Network: http://192.168.0.5:5173`).

Edit `mobile/capacitor.config.ts`, uncomment:
```ts
server: {
  url: 'http://192.168.0.5:5173',
  cleartext: true,
},
```

Terminal 2 — sync + run:
```powershell
cd Z:\projects\voicescape\mobile
npm run sync
npm run open  # Android Studio, then Run
```

### Option B: Build + sync (no live reload, closer to release)

```powershell
cd Z:\projects\voicescape\mobile
npm run sync       # builds client + copies dist into android/
npm run open       # opens Android Studio, hit Run
```

## Building APK

### Debug APK (sideload-friendly)

```powershell
cd Z:\projects\voicescape\mobile
npm run build:apk:debug
```

Output: `mobile/android/app/build/outputs/apk/debug/app-debug.apk`

Install on device:
```powershell
adb install -r mobile/android/app/build/outputs/apk/debug/app-debug.apk
```

### Release APK (signed)

1. Generate a keystore (one-time):
   ```powershell
   mkdir mobile\android\keystores
   keytool -genkey -v -keystore mobile\android\keystores\release.keystore `
           -alias voicescape -keyalg RSA -keysize 2048 -validity 10000
   ```
   Remember the password — losing it means you can never update the app
   on Play Store.

2. Add to `mobile/android/key.properties` (gitignored):
   ```
   storeFile=keystores/release.keystore
   storePassword=...
   keyAlias=voicescape
   keyPassword=...
   ```

3. Edit `mobile/android/app/build.gradle` `signingConfigs.release`:
   ```groovy
   signingConfigs {
     release {
       def keystoreProps = new Properties()
       keystoreProps.load(new FileInputStream(file('../key.properties')))
       storeFile file(keystoreProps['storeFile'])
       storePassword keystoreProps['storePassword']
       keyAlias keystoreProps['keyAlias']
       keyPassword keystoreProps['keyPassword']
     }
   }
   ```

4. Build:
   ```powershell
   npm run build:apk:release
   ```

Output: `mobile/android/app/build/outputs/apk/release/app-release.apk`

## Capabilities matrix

| Feature | Desktop (Electron) | Mobile (Capacitor) |
|---|---|---|
| 録音 | MediaRecorder + fs | MediaRecorder + Filesystem plugin |
| 文字起こし | Deepgram / Whisper API | 同左 (HTTP) |
| 要約 | OpenAI/Claude/Gemini API | 同左 |
| インフォグラフィック (gpt-image-2) | OpenAI 直 (要組織認証) | Worker 経由 (運営側で認証済) |

| 通知 | Electron Notification | Capacitor Notifications |
| エクスプローラ起動 | ✓ shell.openPath | ✗ → 「ギャラリーで開く」に置換 |
| ローカル LLM | Ollama 直 (localhost:11434) | Ollama 経由 (PC/NAS の LAN/Tailscale + 外部アプリ) |
| ファイル保存 | ローカル fs | Filesystem plugin → Documents/ |
| 写真ライブラリ保存 | ✗ | Filesystem + MediaStore |
| 共有シート | ✗ | Capacitor Share |

## Local processing on mobile

Per project mandate: **the local-processing path is preserved on mobile.**
Settings → 「ローカル処理エンドポイント」で以下のいずれかを設定:

1. **自宅 PC の Ollama (Wi-Fi 経由)**: `http://<PCのLAN IP>:11434`
2. **自宅 PC の Ollama (Tailscale 経由)**: 外出先からも自宅マシン直叩き
3. **NAS の Ollama**: UGREEN DXP2800 上の Docker Ollama
4. **コンパニオンアプリ (Maid / Layla / MLC Chat)**: 端末内 LLM、HTTP 経由
5. **Termux + Ollama**: 端末内、上級者向け

これにより、おまかせプランも自前APIキーモードも、加えて **完全ローカル**
モードも全部一つのアプリで切替可能。

## Troubleshooting

### `Failed to load Vite: net::ERR_CLEARTEXT_NOT_PERMITTED`

In `capacitor.config.ts` set `server.cleartext: true` AND in
`AndroidManifest.xml` `<application>` tag add:
```xml
android:usesCleartextTraffic="true"
```
Only needed for HTTP dev server; production should use HTTPS only.

### `gradlew.bat` fails on first run

Usually a JDK version mismatch. Verify `JAVA_HOME` is set to JDK 17:
```powershell
java -version
```
Should print `openjdk version "17.x.x"`. If not, install Android Studio's
embedded JDK and add it to PATH:
```powershell
$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
```

### Recording stops when screen turns off

You haven't implemented the Foreground Service yet. Until then, recording
will pause on screen-off. See `docs/foreground-service.md` (TBD).
