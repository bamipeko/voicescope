# VoiceScope 機能・操作 完全ガイド

このドキュメントはアプリ全機能を画面・要素単位で言語化したものです。
スクリーンショット撮影や説明書作成のリファレンスとして使えます。

---

## 0. アプリ全体構成

### 画面の基本レイアウト

```
┌──────────────┬─────────────────────────────────┐
│              │                                 │
│  サイドバー   │      メインエリア                │
│  (左, 224px) │      (右, 残り全部)              │
│              │                                 │
│  - ナビ       │  - 各画面のコンテンツ             │
│  - フォルダ   │                                 │
│  - その他     │                                 │
│              │                                 │
│              │                                 │
│              │                                 │
│              │     録音ボタン（右下に常駐 🔴）  │
└──────────────┴─────────────────────────────────┘
```

- **サイドバー**: 画面遷移、フォルダ、件数バッジ
- **メインエリア**: 開いている画面のコンテンツ
- **録音ボタン**: どの画面にいても右下に固定表示
- **テーマ**: ダークモード（`#1e1e20` 基調）

### 画面一覧

| 画面 | URL | 役割 |
|---|---|---|
| ダッシュボード | `/` | 録音一覧・新規アップロード |
| 録音詳細 | `/recordings/:id` | 個別録音の文字起こし・要約・操作 |
| Ask All | `/ask-all` | 複数録音をまたぐAI質問 |
| テンプレート | `/templates` | 要約テンプレート管理 |
| 設定 | `/settings` | 全般設定・APIキー・プラン |
| アーカイブ | `/archive` | 退避した録音 |
| ゴミ箱 | `/trash` | 削除予定の録音 |

## 1. サイドバー（全画面共通）

### 構成（上から順に）

#### A. ロゴ
- アプリ名「VoiceScope」（白、太字）
- サブタイトル「音声文字起こし & AI要約」（グレー、小）

#### B. メインナビ
| アイコン | ラベル | リンク先 |
|---|---|---|
| ⊞ | ダッシュボード | `/` |
| ⊕ | Ask All | `/ask-all` |
| ⊟ | テンプレート | `/templates` |
| ⊡ | 設定 | `/settings` |

- 現在いる画面はハイライト（カード背景＋右端に青い縦バー）
- ホバーで背景うっすら明るく

#### C. フォルダセクション
- 見出し「フォルダ」+ 右端に「+」ボタン（新規作成）
- フォルダがある場合のみ表示
- 各フォルダ:
  - 📁 アイコン（変更可、絵文字）
  - フォルダ名
  - 右端に件数（例: `0`, `3`）

- 「+」を押すと入力欄が開いてフォルダ名入力 → エンター/作成

| アイコン | ラベル | 件数バッジ |
|---|---|---|
| 📦 | アーカイブ | 件数（未表示なら0） |
| 🗑 | ゴミ箱 | 件数（未表示なら0） |

#### E. オフラインモード表示（オフライン時のみ）
- 緑のカード「🔒 オフラインモード / 外部通信なし」
- クリックで設定画面へ

#### F. アップデート通知（新版あり時のみ）
- 青いカード「📦 新しいバージョン v0.X.X / クリックで詳細」

## 2. 録音ボタン（全画面共通）

右下に常駐する **🔴 赤い丸ボタン**（直径56px程度）。

### 状態と動作

| 状態 | 表示 | クリック時 |
|---|---|---|
| 未録音 | 🔴 赤丸 | 録音開始 |
| 録音中 | 🔴 + 周りにパルス（赤い波紋） | 録音停止 |
| 一時停止中 | ⏸ | 再開 |

### 録音中の追加UI（右下、赤丸の上に出現）

- 経過時間表示「`0:35`」
- 一時停止ボタン「⏸」
- ハイライトボタン「⭐」（その瞬間にマーカーを刻む）
- マイク選択ドロップダウン
- 録音モード切替（マイクのみ / マイク+システム音声）

### キーボードショートカット（Electron版のみ）

- `Ctrl+Shift+F8`: 録音開始/停止
- `Ctrl+Shift+Q`: ハイライトマーク

## 3. ダッシュボード画面 (`/`)

### 上部ヘッダー

```
[録音一覧]                      [ファイルをアップロード]
```

- 左: 「録音一覧」見出し（フォルダ選択時はフォルダ名+アイコン）
- 右: **青い「ファイルをアップロード」ボタン**

### 検索・フィルタ行

```
[全体 ▾] [キーワード検索...                ] [すべてのタグ ▾] [すべての重要度 ▾]
[ ] アーカイブ済みも検索対象に含める  ← 検索/フィルタ使用時のみ表示
```

- **左の灰色バッジ**: `全体` / `フォルダ内` / `タグ内`（コンテキスト表示）
- **検索入力**: タイトル＋文字起こし全文を検索
- **タグフィルタ**: ドロップダウン
- **重要度フィルタ**: ★, ★★, ★★★
- **アーカイブ含むチェック**: 検索/タグ/重要度のいずれか使用中のみ出現

### アクティブフィルタバッジ（フィルタ適用時）

```
[📁 仕事 ✕] [🏷 重要 ✕]
```
クリックでフィルタ解除。

### 録音カードのリスト

各カードの構造:

```
┌────────────────────────────────────────────────┬──────────┐
│ [タイトル]  [ステータスバッジ]  [★★☆]            │ 🔒 📦 ✕ │
│ 2026/04/28 11:28  --:--   📄 元ファイル名.txt    │          │
│ [タグ1] [タグ2] [タグ3]                          │          │
└────────────────────────────────────────────────┴──────────┘
```

**左側**:
- タイトル（白、太字、長すぎる場合は省略）
- ステータスバッジ（後述）
- 重要度（黄色の星、クリックで 1→2→3→1 ループ）
- 2行目: 日時、長さ、（テキストアップロードなら）📄 元ファイル名
- 3行目: タグ（タグカラーの薄い背景＋濃い文字）

**右側のアクション**（ホバーで出現）:
- 🔒/🔓: ローカル保護トグル（横断検索の対象外にする）
- 📦: アーカイブへ移動
- ✕: ゴミ箱へ移動

### ステータスバッジ

| ステータス | バッジ色 | 意味 |
|---|---|---|
| `アップロード済` | 灰色 | 受信完了、処理待ち |
| `文字起こし中` | 黄パルス | Deepgram/Whisper等で処理中 |
| `文字起こし完了` | 青 | テキスト化完了 |
| `整形中` | 紫パルス | フィラー除去・誤字修正中 |
| `要約中` | 黄パルス | LLMで要約中 |
| `完了` | 緑 | 全処理終了 |
| `エラー` | 赤 | 何らかの失敗 |

### ファイルアップロードフロー

「ファイルをアップロード」または **画面にドラッグ&ドロップ** で:

1. 対応形式の場合のみ受け付け（mp3/wav/m4a/webm/ogg/flac/txt/md）
2. ポップアップ表示:
   ```
   アップロード設定
   ─────────────────
   📄 ファイル名.mp3
   ☑ 自動要約を実行する
   ┌ テンプレート ▾ ┐  ┌ 粒度 ▾ ┐
   │ デフォルト    │  │ 通常   │
   ┌ プロバイダ ▾ ┐  ┌ モデル ▾ ┐
   │ デフォルト    │  │ デフォルト│
   ─────────────────
                [キャンセル] [アップロード]
   ```
3. 「アップロード」 → 自動でパイプライン処理開始

### ドラッグ&ドロップ対応

画面外からファイルをドラッグすると:
- 半透明の青いオーバーレイ表示
- 中央に「ここにファイルをドロップ」メッセージ

### エラー録音について

タイトルが `[Error] xxx` で始まり、ステータスバッジが赤い「エラー」のもの。
クリックで詳細画面へ → 「再処理」ボタンで再実行可能。

## 4. 録音詳細画面 (`/recordings/:id`)

最も多機能な画面。ヘッダー、プレーヤー、4つのタブで構成。

### ヘッダー

```
[← 録音一覧に戻る]                  [↓エクスポート] [📁エクスプローラで開く] [削除]
[タイトル                                                                  ]
[ステータスバッジ] [★★☆] [2026/04/28 11:28]
```

- **左上**: 「← 録音一覧に戻る」リンク
- **タイトル**: 大きく白く。クリックで編集可能（インライン編集）
- **ステータス・重要度・日時**

**右上ボタン群**:

| ボタン | 動作 |
|---|---|
| ↓ エクスポート ▾ | ドロップダウン: すべてMD / 音声DL / 文字起こしテキスト / 要約MD / AI質問履歴 |
| 📁 エクスプローラで開く ▾ | ドロップダウン: 音声ファイルの場所 / データフォルダ全体 |
| 削除 | 確認ダイアログ → ゴミ箱へ |

### エラーバナー（status='error' のとき）

```
┌────────────────────────────────────────────────┐
│ 処理エラー: パイプライン処理中にエラーが発生… [再処理]│
└────────────────────────────────────────────────┘
```

### 音声プレーヤー

```
┌────────────────────────────────────────────────┐
│ ▶  0:00 ━━━━━━━━━━━━━━━━━━━━━━━━━━━ 🔊       │
└────────────────────────────────────────────────┘
```

- 標準的なオーディオプレーヤー
- 文字起こしのセグメントクリックで該当時刻にジャンプ可能

### タブ構成

```
[文字起こし] [要約 (1)] [タグ (3)] [AI質問]
```

タブ右の数字は「この録音内に存在する数」。

### 4-A. 「文字起こし」タブ

#### ツールバー

```
[整形済み][原文]  [整形実行]  ☑ 自動スクロール  [コピー]  [再実行 ▾]
```

| 要素 | 動作 |
|---|---|
| 整形済み/原文トグル | 表示モード切替（処理未完なら「整形中...」表示） |
| 整形実行 | 整形が未実行の場合に手動トリガー |
| 自動スクロール | 音声再生中、現在の発話セグメントへ自動追従 |
| コピー | 全文をクリップボードへ |
| 再実行 ▾ | 別エンジンで再文字起こし（後述） |

#### 「再実行 ▾」ドロップダウン（v0.14.3以降）

```
┌──────────────────────────────────────┐
│ エンジンを選んで「実行」ボタンを押してください │
│ 現在: ⚡ Grok STT ($0.10/h)              │
├──────────────────────────────────────┤
│ ✓ デフォルト（設定で選択したエンジン）       │
├──────────────────────────────────────┤
│   Deepgram (Nova-2)                  │
│   ⚡ Grok STT ($0.10/h)（現在）         │
│   OpenAI Whisper                     │
│   whisper.cpp (ローカル)              │
│   faster-whisper (ローカル)           │
├──────────────────────────────────────┤
│        [ ▶ 実行 ]                    │
└──────────────────────────────────────┘
```

- エンジンクリックで✓マーク（選択のみ、実行はまだ）
- 「▶ 実行」で実際にトリガー
- 処理中はボタン全体 disabled に

#### 話者ラベル行

```
[● speaker_0 ▾] [● speaker_1 ▾]
```

- 各話者の色別ドット
- クリック→ドロップダウンで「○○さん」など実名に変更
- 入力時に既知話者リストから候補表示
- 一度入力した名前は他の録音でも候補に出る

#### 範囲選択行

```
要約範囲を選択  すべて選択             💡 Shift+クリックで範囲選択
```

- セグメント左端のチェックボックスで個別選択
- Shift+クリックで範囲選択
- 選択するとボタンに件数表示
- 要約タブで「指定範囲のみ」要約可能

#### セグメント一覧

各セグメント:

```
☐ 0:00  ● speaker_0
        ど う で す か あ の 連 絡 が ち ょ っ と た ま に
```

- 左: チェックボックス
- 時刻（クリックで音声ジャンプ）
- 話者ドット+名前（クリックで話者編集）
- 本文（クリックでインライン編集→Enter保存）
- 再生中のセグメントは背景色で強調
- ホバーで微妙に背景明るく

### 4-B. 「要約」タブ

#### ツールバー

```
[テンプレート ▾] [粒度 ▾] [プロバイダ ▾] [モデル ▾] [☐ 指定範囲のみ (5件)] [要約生成]

│ LLMに送ります...                       │
├──────────────────────────────────────┤
│ ┌────────────────────────────────┐  │
│ │ 例: 「予算」に関する話題だけを抽出  │  │
│ │ して、箇条書きでまとめてください    │  │
│ │                                │  │
│ └────────────────────────────────┘  │
├──────────────────────────────────────┤
│         [キャンセル] [この指示で要約する]  │
└──────────────────────────────────────┘

複数回要約した場合、横一列にタブ表示:

## 5. Ask All 画面 (`/ask-all`)

複数録音にまたがる質問。

### スコープ選択

```
スコープ:
[ ] フォルダ: [仕事 ▾]
[ ] タグ:    [重要 ▾]
[ ] 重要度:  [★★以上 ▾]
[ ] 全体（直近100件）
☐ ローカル保護録音も含める  ← 通常はOFF
```

### チャット領域

通常のチャットUI。質問→回答。回答下に **「参照された録音:」** リスト（クリックで個別ページへ）。

### 動作仕組み（裏側）

```
1. スコープに合致する録音の要約を全部集める
2. LLMに「どの録音が関連するか」をJSON配列で問う
3. 関連録音（最大5件）の文字起こし全文を投入
4. 詳細回答を生成
5. 履歴に保存
```

### 履歴管理

- セッション切替UI（過去のセッション一覧）
- 「履歴をクリア」ボタン

## 7. 設定画面 (`/settings`)

セクションが多いので折りたたまずスクロール式。

### 7-A. プラン (Subscription)

- 現在のプランをバッジ表示
- 自前APIキー / トライアル / Pro / Heavy / Free+
- 「トライアルコード」入力欄
- アップグレード案内（近日公開）

### 7-B. 動作モード

```
○ オフライン      （ローカル文字起こし＋ローカルLLM）
● ownkey         （自前APIキーで全機能）
○ マネージド     （運営APIキー）
```

選択時に確認ダイアログ。

### 7-C. ローカル文字起こし & LLM

各項目に状態ドット (●緑=利用可 / ●灰=未設定):

- whisper.cpp - モデル一覧（tiny/base/small/medium/large-v3）と DL/DL済みボタン
- Ollama - インストール済みモデル一覧、デフォルトモデル選択、新モデルPull
- カスタムエンドポイント (OpenAI互換):
  - ベースURL入力
  - モデル名入力
  - APIキー入力（任意）
  - 「疎通テスト」ボタン
  - 「? 接続方法のヘルプ」リンク → 5ツール用展開ヘルプ
- faster-whisper - ステータス表示のみ

### 7-D. APIキー（クラウドモード）

各サービスごとに:
- ラベル（Deepgram / OpenAI / Gemini / Grok / Claude）
- 状態（保存済み / 環境変数 / 未設定）
- 入力欄 + 保存ボタン
- マスク表示 + 表示切替

### 7-E. 文字起こし

```
デフォルトエンジン: [Deepgram (Nova-2) ▾]
                  - ⚡ Grok STT — 新・低価格
                  - OpenAI Whisper

## 10. 主な操作フロー

### フロー A: 録音 → 自動処理

```
1. 🔴 録音ボタンクリック
2. マイク許可ダイアログ → 許可
3. 話す
4. 🔴 もう一度クリック → 停止
5. 自動的にダッシュボードへ追加（status='アップロード済'）
6. パイプライン開始:
   - 文字起こし中 (黄パルス)
   - 文字起こし完了 (青)
   - 整形中 (紫パルス) ← 並列実行
   - 要約中 (黄パルス)  ← 並列実行
   - 完了 (緑)
7. 自動でタイトル生成 + タグ付け
8. クリックで録音詳細へ
```

### フロー B: ファイルアップロード

```
1. ダッシュボード「ファイルをアップロード」または D&D
2. 設定ダイアログでテンプレート/モデル選択
3. アップロード → パイプライン自動実行
```

### フロー C: 別エンジンで再文字起こし（比較）

```
1. 録音詳細 → 文字起こしタブ
2. 「再実行 ▾」をクリック
3. ドロップダウンからエンジン選択（✓マーク移動）
4. 「▶ 実行」ボタンで開始
5. 処理完了後、再度「整形→要約」が走る
6. 履歴は transcriptions テーブルに残る
```

### フロー D: 削除 → ゴミ箱 → 自動削除

```
1. ダッシュボード「✕」クリック
2. 確認ダイアログ「ゴミ箱に移動します...」
3. ゴミ箱へ移動（実体は残る）
4. 設定の保持日数（14日デフォルト）経過
5. サーバ起動時に自動削除
   - 完全削除モード: DB+音声すべて消去
   - 音声のみモード: 音声削除→アーカイブへ自動移動
```

## 11. アイコン・色の意味

### ステータス色

| 色 | 用途 |
|---|---|
| 🔵 青 | 進行可能・情報 |
| 🟢 緑 | 完了・成功・利用可 |
| 🟡 黄 | 処理中（パルスアニメ） |
| 🟣 紫 | 整形中（特別ステータス） |
| 🔴 赤 | エラー・削除 |
| ⚪ 灰 | 待機・非アクティブ |

### 主なアイコン

| アイコン | 意味 |
|---|---|
| 🔴 | 録音 |
| 📁 | フォルダ・エクスプローラ |
| 📦 | アーカイブ |
| 🗑 | ゴミ箱・削除 |
| 🔒 / 🔓 | ローカル保護 |
| ⭐ | 推奨・重要度 |
| ⚡ | 高速・新サービス（Grok等） |
| ✓ / ✕ | 選択 / クローズ |
| ↩ | 復元・戻す |
| ▶ | 実行・再生 |
| ▾ | ドロップダウン展開 |

### トースト通知

画面左下に出現するメッセージ（5〜8秒で消える）:

- 緑背景: 成功（保存・完了）
- 黄背景: 警告（フォールバック発生など）
- 赤背景: エラー
- 通常: 情報

## 12. データの保存場所
