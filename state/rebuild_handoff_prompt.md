## 現状理解

VoiceScope は、音声録音、文字起こし、AI要約、録音管理、インフォグラフィック生成を持つローカルファースト志向のアプリです。

現在の主な構成:

- PC版: Electron + React/Vite + Express + sql.js SQLite
- UI: `client/`
- API/処理: `server/`
- Electron: `electron/`
- モバイル: `mobile/` に Capacitor 方向の scaffold があるが、実用レベルでは未完成
- Managed API構想: `worker/` に Cloudflare Worker がある
- データ保存: PC版は `%APPDATA%\VoiceScope\data\voicescope.db` と audio フォルダ中心

重要な思想:

- ローカルファーストを捨てない
- 録音・文字起こし・要約データをユーザーの手元に残せることが強み
- 非エンジニアの友人にも渡せるアプリにしたい
- APIキー設定やローカルLLM設定でユーザーを迷わせたくない
- クラウド依存だけの設計にはしたくない

## 最重要課題

特に重視しているのは次の2点です。

1. スマホ対応ができていない
   - 現状はPC版が中心
   - mobile/ はあるが、実用できるAndroid/iOSアプリとしては未完成
   - スマホで録音し、文字起こしデータや要約をローカル保存できる形にできれば、かなり強い差別化になる

2. ローカル処理でエラーが起きやすい
   - whisper.cpp / faster-whisper / Ollama / ローカルエンドポイントなど、方向性は良いが複雑
   - 非エンジニアが安心して使うには、セットアップ、失敗時の回復、データ保存、モデル管理を再設計する必要がある
   - 「動く人には動く」ではなく、「普通の人が迷わず使える」に近づけたい

## レビュー方針

以下の姿勢で見てください。

- 既存実装を守ることを目的にしない
- 互換性よりも、長期的に勝てる設計を優先してよい
- ただし既存データ、録音、文字起こし、要約を失う提案はNG。移行方針を必ず考える
- 小さな改善点リストで終わらせない
- 「このまま継ぎ足すべきか」「一部作り直すべきか」「全面的に再構成すべきか」を判断する
- 最新のAIモデル、スマホ開発、ローカルAI、オンデバイス/ローカル保存の現実的な選択肢を踏まえる
- 公式ドキュメントや現在の制約を確認し、古い前提で判断しない

## 特に評価してほしい論点

1. スマホアプリ化の正攻法
   - Capacitor継続でよいか
   - React Native / Expo / Flutter / PWA / ネイティブ実装のどれがよいか
   - 録音、バックグラウンド録音、ローカルファイル保存、SQLite、通知、権限、音声アップロードの観点で比較
   - iOS/Android両対応を見据えた場合の現実解

2. データ保存設計
   - スマホでローカル保存するなら、SQLite / Filesystem / IndexedDB / OPFS / Capacitor SQLite など何がよいか
   - PC版とスマホ版でデータ形式を揃えるべきか
   - エクスポート、バックアップ、端末移行、復元をどう設計すべきか
   - 既存の `voicescope.db` からどう移行するか

3. ローカルAI処理の再設計
   - PCローカル、スマホローカル、NAS/自宅PCサーバ、クラウドAPIの役割分担
   - whisper.cpp / faster-whisper / Ollama をどう扱うべきか
   - スマホ上で完結させるべき処理と、PC/NASに逃がすべき処理の切り分け
   - ローカル処理失敗時のフォールバック設計

4. アーキテクチャ再構成
   - 現在の Electron + Express + React + sql.js を続けるべきか
   - APIサーバを持つ構成がスマホ展開に向いているか
   - client/server/electron/mobile/worker が複雑化していないか
   - 共有ロジック、データ層、同期層、AIプロバイダ層をどう分け直すべきか

5. UX再設計
   - 非エンジニアでも使える録音アプリにするには何を削るべきか
   - APIキー、ローカルモデル、プラン、要約テンプレート、エクスポートをどう見せるべきか
   - 「録音する」「文字起こしを見る」「要約する」「保存/共有する」の中心導線をどう単純化するか

6. 事業・差別化
   - スマホでローカル保存できる録音/文字起こし/要約アプリとして、何が強みになるか
   - クラウドAIのみの競合とどう差別化できるか
   - ローカルファーストとおまかせクラウドの両立をどう設計するか
